2. Backend: How “Login Successful” Is Detected

The backend determines success using verifiable, server-side signals.

2.1 Primary Indicators (Strong Signals)

Any ONE of the following is sufficient:

Access token issued

OAuth access token / session token exists

Token is non-empty and unexpired

Session established

Session ID stored server-side

User context attached to session

Authenticated API call succeeds

/profile, /me, /accounts, etc. returns 200 OK

No 401 / 403

2.2 Example (Java / Spring-style)
boolean isLoginSuccessful() {
    return tokenStore.hasValidAccessToken()
        && authApi.getProfile().getStatus() == 200;
}


This is backend detection, not UI guessing.

3. Frontend: How It Knows Login Succeeded

The frontend never decides success on its own.

3.1 Frontend Flow (Correct)

Submit credentials / OTP

Call backend /auth/login

Backend responds with:

200 + token/session → success

401 / 403 → failure

202 → pending (OTP / CAPTCHA)

3.2 Example Response Contract
{
  "status": "SUCCESS",
  "authenticated": true,
  "redirectUrl": "/dashboard"
}

3.3 Frontend Logic
if (response.authenticated === true) {
    window.location.href = response.redirectUrl;
}


Redirection is frontend behavior, but decision is backend-validated.

4. Redirect to Dashboard — Who Controls It?
Concern	Owner
Authentication validation	Backend
Token/session creation	Backend
Redirect trigger	Frontend
Redirect target	Backend-defined
Access protection	Backend middleware

The frontend redirects only after backend confirmation.

5. Backend Must Also Enforce Dashboard Access

Even after redirect, the backend must re-validate authentication.

5.1 Dashboard API Guard
@GetMapping("/dashboard")
public ResponseEntity<?> dashboard(HttpSession session) {
    if (!session.isAuthenticated()) {
        return ResponseEntity.status(401).build();
    }
    return ResponseEntity.ok(dashboardData);
}


This prevents:

Manual URL access

Stale sessions

Token replay

6. Automation / Selenium Context (Your Use Case)

If this is automated login (Selenium / WebDriver):

6.1 DO NOT rely on:

Page title change

URL change alone

DOM element visibility alone

6.2 Correct Detection Strategy

Backend-validated signals via UI:

Capture token from:

Network tab (XHR response)

LocalStorage / Cookies

Call a protected API using that token

Verify 200 OK

6.3 Example
String token = browser.getLocalStorage("access_token");

boolean success = apiClient
    .withToken(token)
    .getProfile()
    .is200();


Only then treat login as successful.

7. Summary (Direct Answer)

Login success detection is a BACKEND feature.
Redirection to dashboard is a FRONTEND action, triggered only after backend confirmation.

Correct Architecture

Backend decides: Authenticated or not

Frontend reacts: Redirect or block

Backend enforces: Dashboard access control
