DASHBOARD PAGE – COMPLETE IMPLEMENTATION GUIDE
1. DASHBOARD PURPOSE (NON-NEGOTIABLE)

The Dashboard is NOT a login page and NOT a trading page.

It is a real-time operational command center that answers, at a glance:

Is the system authenticated?

Is PRIMARY access token valid?

Is market data flowing?

Is risk under control?

Is the system safe to trade right now?

If any of these are NO, the dashboard must clearly show it.

2. DASHBOARD ENTRY CONDITIONS (HARD GATE)

The backend must allow dashboard access only if:

Condition	Required
PRIMARY token exists	YES
PRIMARY token valid	YES
Auth state	AUTH_CONFIRMED
Cooldown	INACTIVE
DB health	READABLE
Backend Guard
GET /dashboard
→ 401 if PRIMARY missing or expired


The frontend never bypasses this.
